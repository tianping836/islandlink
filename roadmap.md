# 开发路线图（详细版）

> 军哥的时间按周排程。每个 Phase 标注了预计周数和核心交付物。

---

## Phase 0：原型验证（第 1-2 周）

**目标：不写代码，验证需求**

### 任务清单
- [ ] 飞书多维表格搭建数据原型
  - 创建"人脉"表（姓名/角色/单位/电话/备注）
  - 创建"案件"表（名称/案号/类型/法院/状态/摘要）
  - 创建"关联"表（人脉→案件←关联，含角色字段）
  - 设置关联列
  - 手动录入 10+ 案件 + 30+ 人脉
- [ ] 每天录入当日工作涉及的人和案，感受：
  - 字段够不够？
  - 关联列好用不好用？
  - 哪里让你觉得"这里应该有更好的方式"？
- [ ] 截图记录飞书的交互，尤其是关联列的搜索、添加、展示
- [ ] 下载 WOLB 并录入 10 条真实数据
  - 对比：WOLB 的"事件-参与人"关联 vs 飞书的"关联列"
  - 记录 WOLB 做得好的地方
- [ ] 决定 App 名字（暂时用 CaseNetwork，你可以改）

### 交付物
- 数据模型 v1（验证后的字段清单）
- 痛点列表（飞书/WOLB 哪里不够用）
- 设计参考截图集

## Phase 1：核心 MVP — iOS（第 3-10 周）

**目标：iOS 上能跑的核心功能**

### Sprint 1：项目搭建 + 数据模型（第 3-4 周）
- [ ] Xcode 项目创建（iOS Target，最低 iOS 17）
- [ ] SwiftData 数据模型实现（Person / Case / CasePerson / CaseEvent / Tag）
- [ ] ModelContainer 配置（先本地，不接 CloudKit）
- [ ] 单元测试：数据创建、关联、查询
- [ ] 预设数据（PreviewData）用于 UI 开发
- [ ] 拼音扩展实现和测试

### Sprint 2：人脉 CRUD（第 5-6 周）
- [ ] 人脉列表页（SwiftUI List）
- [ ] 搜索（名字/拼音/单位）
- [ ] 按角色筛选（分段控件）
- [ ] 人脉新建/编辑表单
- [ ] 人脉详情页（基本信息 + 关联案件列表）
- [ ] 删除确认 + 滑动删除
- [ ] 空状态

### Sprint 3：案件 CRUD + 关联（第 7-8 周）
- [ ] 案件列表页（按状态分段）
- [ ] 案件新建/编辑表单
- [ ] 案件详情页
- [ ] **添加参与人弹窗**（搜索+选择+角色设定）← 核心
- [ ] 案件详情页参与人列表（按角色分组展示）
- [ ] 参与人名字点击 → 跳转人脉详情
- [ ] 从人脉详情跳转案件详情（导航不丢上下文）

### Sprint 4：全局搜索 + 打磨（第 9-10 周）
- [ ] 全局搜索（同时搜人+案件）
- [ ] TabBar 导航
- [ ] UI 美化（第一轮）
- [ ] 基础测试 + Bug 修复
- [ ] 在真机上跑通

### 交付物
- 可在 iPhone 上运行的 App（TestFlight 可用）
- 核心功能完整：人脉管理、案件管理、双向关联、搜索

## 每周固定动作

不管在哪个 Phase，每周做这些：

1. **周一**：Review 上周进度，调整本周计划
2. **每天**：至少写 30 分钟代码（保持节奏）
3. **周五**：在真机上跑一遍当天成果，录屏留档
4. **每完成一个 Sprint**：写一段总结到 `memory/topics/case-network-app.md`
